# Command Memory Effects

How each Alex command modifies the `.github/` cognitive architecture files.

## Command Reference

| Command                    | Trigger                              | Memory Files Modified                                                     | Details                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| -------------------------- | ------------------------------------ | ------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Initialize**             | `sync-architecture.cjs` or extension | ALL `.github/`                                                            | Copies entire architecture from extension package: `copilot-instructions.md`, `hooks.json`, `instructions/`, `prompts/`, `config/`, `agents/`, `skills/`, `muscles/`, `assets/`. Creates empty `episodic/` folder. Writes `config/alex-manifest.json` with checksums. Detects project persona and writes `config/user-profile.json`.                                                                                                             |
| **Reset**                  | Extension reset command              | ALL `.github/` (destructive)                                              | Deletes `copilot-instructions.md`, `hooks.json`, `instructions/`, `prompts/`, `episodic/`, `domain-knowledge/`, `config/`, `agents/`, `skills/`, `muscles/`, `assets/`. Then runs full Initialize. All user-created episodic records and profile data are lost.                                                                                                                                                                                  |
| **Upgrade**                | `sync-architecture.cjs` or extension | ALL `.github/` (preserving)                                               | 1) Backs up entire `.github/` to `archive/upgrades/`. 2) Deletes `.github/`. 3) Fresh install from extension. 4) Migrates Active Context fields from backup to new `copilot-instructions.md`. 5) Auto-restores: user-created skills, `user-profile.json`, `episodic/` records. 6) Normalizes all `skills/*/synapses.json` to current schema. 7) Runs Dream Protocol post-upgrade.                                                                |
| **Dream**                  | Dream prompt, `brain-qa.cjs`         | `episodic/dream-report-{date}-{timestamp}.md`                             | Scans all memory files for broken synapse references. Auto-repairs where possible (consolidated file mappings). Writes a dream report to `episodic/`. Does NOT modify source files -- only reads and reports.                                                                                                                                                                                                                                    |
| **Deep Brain QA**          | `Alex: Deep Brain QA`                | `episodic/dream-report-{date}-{timestamp}.md`                             | Runs Dream Protocol in silent mode. Same file effects as Dream -- writes a report to `episodic/`.                                                                                                                                                                                                                                                                                                                                                |
| **Self-Actualize**         | Selfactualize prompt                 | `copilot-instructions.md`, `episodic/self-actualization-{date}.prompt.md` | Phase 1: Scans synapse health (read-only). Phase 2: Checks version consistency (read-only). Phase 3: Assesses memory architecture counts (read-only). Phase 4: Auto-promotes domain knowledge to Global Knowledge repo (writes to GK repo if configured). Phase 5: Generates recommendations. Phase 6: Writes full session record to `episodic/`. Phase 7: Updates `Last Assessed` field in Active Context section of `copilot-instructions.md`. |
| **Meditate**               | Meditate prompt                      | None directly                                                             | Opens chat panel in meditation mode. The LLM conversation that follows is expected to create/update memory files (`episodic/`, `instructions/`, `skills/`), but the command itself only sets UI state. Offers buttons to trigger Self-Actualize and Dream.                                                                                                                                                                                       |
| **North Star**             | North star prompt                    | None directly                                                             | Opens chat panel with `/northstar` prompt. The LLM conversation may update `copilot-instructions.md` Active Context fields (North Star, Guidelines), but the command itself only opens chat.                                                                                                                                                                                                                                                     |
| **Profile**                | Profile prompt                       | None directly                                                             | Displays current `config/user-profile.json` contents. When user provides updates via conversation, the LLM updates the profile. The command itself only reads and displays.                                                                                                                                                                                                                                                                      |
| **Personality Mode**       | Welcome View toggle                  | `copilot-instructions.md`                                                 | Updates the `Tone:` field in Active Context section via `activeContextManager`. Values: `_(auto)_`, `Concise, code-first`, or `Explanatory, conversational`.                                                                                                                                                                                                                                                                                     |
| **Persona Detection**      | Automatic on init/meditate           | `config/user-profile.json`, `copilot-instructions.md`                     | Detects project type from workspace files. Writes `projectPersona` object to `user-profile.json` (id, confidence, detectedAt, reasons). Updates `Persona:` field in Active Context of `copilot-instructions.md`. Merges detected technologies into `primaryTechnologies` array.                                                                                                                                                                  |
| **Generate Skill Catalog** | `gen-skill-catalog.cjs` or extension | None in `.github/`                                                        | Reads `skills/*/SKILL.md` and generates `alex_docs/skills/SKILLS-CATALOG.md` (outside `.github/`).                                                                                                                                                                                                                                                                                                                                               |

## File-Centric View

Which commands write to each file/directory:

| File/Directory              | Written By                                                                      | Operation                                                                                                                |
| --------------------------- | ------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------ |
| `copilot-instructions.md`   | Initialize, Reset, Upgrade, Self-Actualize, Personality Mode, Persona Detection | Deploy (init/reset/upgrade), update Active Context fields (self-act/persona/tone)                                        |
| `hooks.json`                | Initialize, Reset, Upgrade                                                      | Deploy only                                                                                                              |
| `instructions/`             | Initialize, Reset, Upgrade                                                      | Deploy only (entire directory copied from extension)                                                                     |
| `prompts/`                  | Initialize, Reset, Upgrade                                                      | Deploy only                                                                                                              |
| `skills/`                   | Initialize, Reset, Upgrade                                                      | Deploy from extension; Upgrade also restores user-created skills from backup                                             |
| `skills/*/synapses.json`    | Upgrade                                                                         | Normalizes schema (string strengths to numeric, `synapses[]` to `connections[]`, adds `$schema`/`skillId`/`inheritance`) |
| `agents/`                   | Initialize, Reset, Upgrade                                                      | Deploy only                                                                                                              |
| `muscles/`                  | Initialize, Reset, Upgrade                                                      | Deploy only                                                                                                              |
| `assets/`                   | Initialize, Reset, Upgrade                                                      | Deploy only                                                                                                              |
| `config/alex-manifest.json` | Initialize, Reset, Upgrade                                                      | Created with version, timestamp, and file checksums                                                                      |
| `config/user-profile.json`  | Initialize, Upgrade, Persona Detection                                          | Created on init; restored from backup on upgrade; updated with persona and technologies                                  |
| `episodic/`                 | Initialize, Reset, Upgrade, Dream, Deep Brain QA, Self-Actualize                | Created empty on init; restored from backup on upgrade; dream reports and self-actualization records written here        |
| `domain-knowledge/`         | Reset (deleted)                                                                 | Legacy directory -- deleted during reset, not recreated                                                                  |

## Active Context Fields

The Active Context section in `copilot-instructions.md` is managed by `activeContextManager.ts`. Only specific fields are written programmatically:

| Field            | Written By              | When                                                                 |
| ---------------- | ----------------------- | -------------------------------------------------------------------- |
| `Persona:`       | Persona Detection       | On init (if `updateSlots: true`), on meditate, on chat session start |
| `Tone:`          | Personality Mode toggle | When user changes personality mode in Welcome View                   |
| `Last Assessed:` | Self-Actualize          | At end of self-actualization protocol (Phase 7)                      |
| All 10 fields    | Upgrade (migration)     | Non-placeholder values from backup are migrated to fresh install     |

Fields that are NOT written programmatically (LLM-managed only): `Objective`, `Focus Trifectas`, `Priorities`, `Principles`, `Recent`, `North Star`, `Guidelines`.

## Safety Guards

| Guard                            | Protects Against                                                                  |
| -------------------------------- | --------------------------------------------------------------------------------- |
| **MASTER-ALEX-PROTECTED.json**   | Init/Reset/Upgrade on Master workspace (kill switch with double-confirm override) |
| **Operation Lock**               | Concurrent command execution (only one architecture command runs at a time)       |
| **Cognitive Level Gate**         | Dream, Self-Actualize, Meditate, North Star require minimum cognitive level       |
| **Active Context Mutex**         | Concurrent writes to `copilot-instructions.md` (serialized via async mutex)       |
| **Master Focus Trifectas Block** | `updateActiveContext` refuses to overwrite Focus Trifectas on Master workspace    |
