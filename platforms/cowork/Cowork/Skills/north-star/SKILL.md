---
name: North Star
description: Define and align to a clear project vision, quality standards, and mission that guides every decision.
---

## Purpose

Every project needs a clear direction. Without one, effort scatters across competing priorities and decisions become political instead of strategic. This skill helps define, communicate, and align work to a clear North Star: the mission that shapes every choice.

## Steps

1. Define the North Star: what is the single most important outcome this project exists to achieve?
2. Translate the vision into measurable quality standards
3. Create alignment tests that evaluate whether any proposed work supports the mission
4. Communicate the North Star in every status update, decision, and planning session
5. Revisit quarterly: is the North Star still right?

## North Star Definition Template

| Element           | Description                                     | Example                                                                 |
| ----------------- | ----------------------------------------------- | ----------------------------------------------------------------------- |
| Mission statement | One sentence describing why this project exists | "Reduce customer onboarding time from 14 days to 2 days"                |
| Success metrics   | 2-3 measurable outcomes                         | Onboarding time, customer satisfaction score, support ticket volume     |
| Not in scope      | What this project explicitly does NOT do        | "We are not redesigning the product; we are fixing the onboarding flow" |
| Time horizon      | When success will be measured                   | "Measure impact 90 days after launch"                                   |

## Alignment Test

Before approving any work item, feature request, or change, ask:

1. Does this directly support the North Star?
2. If we did NOT do this, would we still reach the North Star?
3. Is this the highest-impact thing we could do right now?

If the answer to question 1 is "no" or question 2 is "yes": defer or decline.

## Quality Standards Framework

Define what "good enough" means for this project:

| Dimension    | Minimum Standard                         | Aspirational Standard                        |
| ------------ | ---------------------------------------- | -------------------------------------------- |
| Accuracy     | Zero factual errors in published content | Independent verification of all claims       |
| Timeliness   | Delivered by agreed deadline             | Delivered early with buffer for review       |
| Completeness | All required sections present            | Anticipates follow-up questions              |
| Clarity      | Understandable by target audience        | Understandable by anyone in the organization |
| Consistency  | Follows established templates            | Sets new standard others want to adopt       |

## Decision Framework

When facing a choice between competing priorities:

1. Which option is closer to the North Star?
2. Which is reversible? (Prefer reversible decisions; decide faster)
3. Which has the best effort-to-impact ratio?
4. What would we regret NOT doing in 6 months?

## Communication Template

Include in every status update or planning session:

**North Star reminder**: [one sentence mission]
**This period's progress toward North Star**: [specific metric movement]
**Biggest risk to the North Star**: [one sentence]
**Biggest opportunity**: [one sentence]

## Output Format

- North Star definition document with mission, metrics, scope, and timeline
- Alignment test criteria for evaluating work items
- Quality standards table customized to the project

## Guidelines

- A North Star that tries to be everything accomplishes nothing. Be specific.
- If you cannot measure it, you cannot know if you reached it
- "But we've always done it this way" is not a reason to keep doing it
- Revisit the North Star when the environment changes, not on a fixed schedule
- The best North Star fits in one sentence that anyone in the organization can repeat
