## Purpose

Executives decide in minutes. This skill transforms complex data into narratives that influence outcomes. The goal is not to present information but to drive action.

Core principle: data tells, stories sell.

## The SCQA Framework

Structure every executive communication:

| Element          | Purpose                  | Example                                                 |
| ---------------- | ------------------------ | ------------------------------------------------------- |
| **S**ituation    | Establish shared context | "Our AI adoption is below industry benchmarks."         |
| **C**omplication | Introduce the tension    | "Without intervention, we risk falling further behind." |
| **Q**uestion     | The problem to solve     | "How do we accelerate AI adoption?"                     |
| **A**nswer       | Your recommendation      | "Implement readiness assessment before deployment."     |

## The Pyramid Principle

Always state the conclusion first, then supporting evidence. For every claim, apply the "So What?" test:

- Bad: "Revenue had a positive trend."
- Good: "Revenue grew 34% YoY but growth decelerated from 22% to 8%. If the trend continues, plan for flat next quarter."

Group supporting points using MECE (Mutually Exclusive, Collectively Exhaustive). Order by importance, time, structure, or priority.

## Three-Act Data Story

| Act        | Purpose                 | Content                       |
| ---------- | ----------------------- | ----------------------------- |
| Setup      | Context and stakes      | Why this matters now          |
| Conflict   | The problem or surprise | What's broken, what's at risk |
| Resolution | Your solution           | What to do, expected outcomes |

## Steps

1. Identify the audience and the decision they need to make
2. Write the Big Idea in one sentence: "[Audience] should [action] because [evidence]"
3. Structure the argument using SCQA or Three-Act framework
4. Select 3 key data points that support the argument (not more)
5. For each data point, write the "so what?" in business language
6. Create a Word document or PowerPoint with the executive brief structure:
   - Headline (1 sentence: the recommendation)
   - Context (2-3 sentences: why now, what's at stake)
   - Key findings (3 bullets max with supporting data)
   - Recommendation (specific action with trade-offs)
   - Ask (what you need from the executive)
   - Next steps (immediate actions with owners and dates)
7. Prepare backup detail in an appendix: methodology, full data tables, risk register

## Numbers That Stick

| Technique  | Example                                                   |
| ---------- | --------------------------------------------------------- |
| Anchoring  | "That's 50% more than last year"                          |
| Humanizing | "Each hour saved = 2,000 employees x $50/hr = $100K/year" |
| Comparison | "The cost of a coffee per employee per day"               |
| Rounding   | "$2.3M" not "$2,347,891.23"                               |

## The Ask

Every executive communication needs a clear ask:

| Type       | Example                                    |
| ---------- | ------------------------------------------ |
| Decision   | "Approve the $500K investment"             |
| Resource   | "Allocate 3 FTEs for 6 months"             |
| Alignment  | "Confirm this direction before we proceed" |
| Escalation | "Remove the blocker with [stakeholder]"    |

## Objection Handling

Anticipate objections and use the Acknowledge-Bridge-Response pattern:

1. Acknowledge: "That's a fair concern..."
2. Bridge: "What we've found is..."
3. Response: "...which is why we recommend..."

## Guidelines

- Lead with the conclusion, never the journey
- One idea per paragraph, one message per slide
- Use "we recommend" not "it is recommended"
- Specific, measurable claims only (no vague adjectives)
- Ruthlessly cut: if a point doesn't support the Big Idea, remove it
- Flag risks and assumptions explicitly
- Keep backup detail available but out of the main narrative
