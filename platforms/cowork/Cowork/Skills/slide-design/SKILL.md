---
name: Slide Design
description: Design impactful presentation slides using visual hierarchy, minimal text patterns, and data visualization principles.
---

## Purpose

Create presentations that communicate, not decorate. Most slides fail because they contain too much text, no visual hierarchy, and no clear takeaway. This skill teaches how to design slides that an audience can absorb in seconds.

## Steps

1. Start with the narrative structure (use executive-storytelling or data-storytelling framework)
2. One message per slide. If a slide makes two points, split it into two slides.
3. Write the slide title as a takeaway sentence, not a topic label:
   - Bad: "Q3 Revenue Results"
   - Good: "Q3 Revenue Grew 34% but Growth Rate Is Slowing"
4. Choose the right visual for the content:
   - Data comparison: bar chart
   - Trend over time: line chart
   - Part of whole: stacked bar or pie (max 5 slices)
   - Process: numbered steps or flow diagram
   - Concept: simple diagram with labeled shapes
   - Quote or key message: large text, minimal decoration
5. Apply the 3-second test: can the audience get the point in 3 seconds?
6. Remove clutter: gridlines, unnecessary legends, decorative elements, logo on every slide
7. Create the PowerPoint using the organization's template if available

## Text Rules

| Rule                         | Rationale                             |
| ---------------------------- | ------------------------------------- |
| Max 6 words in a title       | Forces clarity                        |
| Max 6 bullets per slide      | Prevents wall of text                 |
| Max 6 words per bullet       | Forces concise language               |
| No full sentences in bullets | Bullets are signposts, not paragraphs |
| Font size minimum 24pt       | Readable from back of room            |

## Visual Hierarchy

Direct the eye in this order:

1. Title (the takeaway)
2. Hero visual (chart, diagram, or key number)
3. Supporting detail (annotations, source, footnotes)

Use size, color, and position:

- Bigger = more important
- Color = draws attention (use sparingly; one highlight color)
- Top-left to bottom-right reading flow (in left-to-right languages)

## Slide Types

| Type             | When to use               | Layout                          |
| ---------------- | ------------------------- | ------------------------------- |
| Title slide      | Opening                   | Title + subtitle + speaker name |
| Section divider  | Transition between topics | Short phrase, large text        |
| Insight slide    | Key finding               | Title-as-takeaway + one chart   |
| Comparison slide | Two options               | Side-by-side layout             |
| Big number       | One key metric            | Large number + context sentence |
| Summary/Ask      | Closing                   | 3 bullets + clear ask           |

## Chart Formatting for Slides

- Title = the insight, not the metric name
- Remove gridlines
- Remove chart borders
- Use direct labels instead of legends when possible
- Highlight the key data point with contrasting color
- Gray out non-essential data points
- Source attribution in small text at bottom

## Output Format

- PowerPoint presentation following the organization's template
- Each slide passes the 3-second test
- Speaker notes with talking points for each slide

## Guidelines

- Slides support the speaker; they are not the presentation
- If you can remove an element and the slide still works, remove it
- Never read your slides aloud; the audience can read
- White space is a design choice, not wasted space
- Consistent styling: same fonts, colors, and chart types throughout
- End with a clear ask or call to action, not "Questions?"
